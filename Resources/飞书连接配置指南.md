# 飞书连接配置指南

**创建日期:** 2026-03-01  
**用途:** 将 OpenClaw 接入飞书（Feishu/Lark）  
**适用场景:** 企业微信之外的另一个办公 IM 渠道

---

## 📦 方案选择

### 方案 A: OpenClaw 官方飞书插件 ⭐ 推荐新手

| 优点 | 缺点 |
|------|------|
| ✅ 安装简单，5 分钟搞定 | ⚠️ 媒体文件支持有限 |
| ✅ 与 OpenClaw 深度集成 | |
| ✅ 官方维护，稳定性好 | |

### 方案 B: AlexAnys 社区桥接版 ⭐ 推荐高级用户

| 优点 | 缺点 |
|------|------|
| ✅ 媒体支持完善（图片/视频/文件） | ⚠️ 需要额外维护桥接进程 |
| ✅ 支持飞书和国际版 Lark | ⚠️ 配置稍复杂 |
| ✅ 与 Gateway 进程隔离 | |

## 🚀 方案 A: 官方插件配置

### 步骤 1: 安装飞书插件

```bash
openclaw plugins install feishu
```

### 步骤 2: 创建飞书应用

1. **访问飞书开放平台**：https://open.feishu.cn/
2. **登录企业账号**
3. **进入「应用开发」→「企业自建应用」**
4. **点击「创建应用」**
   - 应用名称：如「AI 助手」
   - 应用图标：可选
   - 点击「创建」

### 步骤 3: 获取凭证

1. 进入应用管理页面
2. 点击「凭证与基础信息」
3. 复制以下信息：
   - **App ID**（格式：`cli_xxxxxxxxxxxxxxxx`）
   - **App Secret**（点击「获取」后复制）

⚠️ **重要：** 立即保存 App Secret，关闭后无法再次查看！

### 步骤 4: 配置应用权限

1. 在应用管理页面，点击「权限管理」
2. 点击「申请权限」
3. 搜索并添加以下权限：
   - ✅ `im:message` - 发送消息
   - ✅ `im:chat` - 聊天管理
   - ✅ `contact:user.base:readonly` - 读取用户信息
4. 点击「提交申请」（管理员审批）

### 步骤 5: 配置机器人

1. 点击「应用功能」→「机器人」
2. 点击「添加机器人」
3. 配置机器人信息：
   - 名称：如「AI 助手」
   - 头像：可选
4. 点击「完成」

### 步骤 6: 配置事件订阅（长连接）

1. 点击「事件与回调」
2. 选择 **「使用长连接接收事件」**
3. 订阅以下事件：
   - ✅ `im.message.receive_v1` - 接收消息
   - ✅ `im.chat.update_v1` - 聊天更新
4. 点击「保存」

### 步骤 7: 配置 OpenClaw

```bash
# 启用飞书通道
openclaw config set channels.feishu.enabled true

# 配置 App ID
openclaw config set channels.feishu.appId "cli_你的 AppID"

# 配置 App Secret
openclaw config set channels.feishu.appSecret "你的 AppSecret"

# 配置域名（国内用 feishu，国际用 lark）
openclaw config set channels.feishu.domain "feishu"

# 配置连接模式（推荐长连接）
openclaw config set channels.feishu.connectionMode "websocket"

# 重启网关
openclaw gateway restart
```

### 步骤 8: 发布应用

1. 回到飞书开放平台应用管理
2. 点击「版本管理与发布」
3. 点击「创建版本」
4. 填写版本信息后提交
5. 管理员审核通过后即可使用

### 步骤 9: 添加到群组

1. 在飞书中打开任意群组
2. 点击右上角「...」→「添加机器人」
3. 选择你的 AI 助手机器人
4. 发送消息测试

## 🔧 方案 B: 社区桥接版

### 安装步骤

```bash
# 1. 克隆仓库
git clone https://github.com/AlexAnys/feishu-openclaw.git
cd feishu-openclaw

# 2. 安装依赖
npm install

# 3. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，填入 App ID 和 App Secret

# 4. 启动桥接服务
npm start
```

### 配置 OpenClaw

编辑 `~/.openclaw/openclaw.json`：

```json
{
  "channels": {
    "feishu": {
      "enabled": true,
      "domain": "feishu",
      "connectionMode": "websocket",
      "appId": "cli_xxx",
      "appSecret": "xxx"
    }
  }
}
```

详细文档：https://github.com/AlexAnys/feishu-openclaw

## ❗ 常见问题

### Q1: 消息收不到？

**检查：**
- 事件订阅是否正确配置
- 权限是否已审批
- 长连接是否已启用

### Q2: 图片无法发送/接收？

**方案：**
- 使用社区桥接版（媒体支持更好）
- 检查 `mediaMaxMb` 配置

### Q3: 国际版 Lark 如何配置？

```bash
openclaw config set channels.feishu.domain "lark"
```

### Q4: 多个飞书应用？

在 `openclaw.json` 中配置多个 accounts：

```json
{
  "channels": {
    "feishu": {
      "accounts": {
        "main": {
          "appId": "cli_xxx1",
          "appSecret": "xxx1"
        },
        "backup": {
          "appId": "cli_xxx2",
          "appSecret": "xxx2"
        }
      }
    }
  }
}
```
