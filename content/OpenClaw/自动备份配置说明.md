# OpenClaw 自动备份配置

> **设置时间：** 2026-03-03  
> **备份频率：** 每周日 凌晨 2:00  
> **存储位置：** GitHub Private Repository

---

## 📋 配置概览

| 项目 | 配置 |
|------|------|
| **备份频率** | 每周一次（周日 2:00 AM） |
| **备份内容** | openclaw.json, device.json, jobs.json, paired.json |
| **存储位置** | `obsidian-docs/content/OpenClaw/` |
| **保留策略** | 保留最近 4 次备份 |
| **执行用户** | admin |
| **日志文件** | `/tmp/openclaw-backup.log` |

---

## 🤖 自动化脚本

### 脚本位置
```
/home/admin/.openclaw/workspace/obsidian-docs/scripts/backup-openclaw-config.sh
```

### Cron 配置
```bash
0 2 * * 0 /home/admin/.openclaw/workspace/obsidian-docs/scripts/backup-openclaw-config.sh >> /tmp/openclaw-backup.log 2>&1
```

**解释：**
- `0 2 * * 0` = 每周日 凌晨 2:00
- 输出重定向到 `/tmp/openclaw-backup.log`

## 📁 备份内容

每次备份会生成一个带日期的 Markdown 文件：

```

OpenClaw-核心配置备份-2026-03-17.md
OpenClaw-核心配置备份-2026-03-24.md
...
```

**包含内容：**
- ✅ `openclaw.json` - 主配置
- ✅ `device.json` - 设备身份
- ✅ `jobs.json` - 定时任务
- ✅ `paired.json` - 配对设备
- ✅ 安全提醒和使用说明

## 🔄 保留策略

自动清理旧备份，**只保留最近 4 次**：
- 减少仓库大小
- 保留一个月的配置历史
- 足够用于回滚和恢复

## 🔧 修改备份频率

编辑 crontab：

```bash
crontab -e
```

**常用配置示例：**

| 频率 | Cron 表达式 |
|------|------------|
| 每天凌晨 2 点 | `0 2 * * *` |
| 每周一凌晨 2 点 | `0 2 * * 1` |
| 每周日凌晨 2 点 | `0 2 * * 0` |
| 每月 1 号凌晨 2 点 | `0 2 1 * *` |

## ⚠️ 安全提醒

1. **私有仓库** - 确保 GitHub 仓库是私有的
2. **不要公开分享** - 配置包含敏感信息
3. **定期轮换密钥** - 建议每 30-90 天更新一次

## 📝 恢复配置

如果需要从备份恢复：

```bash
# 1. 停止 OpenClaw 服务
openclaw gateway stop

# 2. 下载备份文件
cd /home/admin/.openclaw/workspace/obsidian-docs/content/OpenClaw
# 选择需要的备份文件

# 3. 恢复配置
cp OpenClaw-核心配置备份-2026-03-03.md /home/admin/.openclaw/openclaw.json

# 4. 重启服务
openclaw gateway start
```
