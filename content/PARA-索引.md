# PARA 知识管理系统

本文件库采用 **PARA 方法** 组织：

## 📁 文件夹结构

| 文件夹 | 说明 | 文件数 |
|--------|------|--------|
| **Projects** | 正在进行的具体项目，有明确目标和截止日期 | 3 |
| **Areas** | 需要持续关注的责任领域，无明确结束日期 | 2 |
| **Resources** | 未来可能有用的主题或兴趣 | 8 |
| **Archives** | 已完成或不活跃的项目/内容 | 1 |
| **Inbox** | 临时收集，待处理的内容 | 0 |
| **Daily** | 每日笔记/日记 | 0 |

## 🔖 标签系统

### 核心标签
- #项目/进行中 - 活跃项目
- #领域 - 责任领域
- #资源 - 参考资料
- #归档 - 已完成内容
- #收件箱 - 待处理
- #日记 - 每日记录

### 主题标签
- #AI - 人工智能相关
- #域名 - 域名相关
- #微信机器人 - 企业微信/微信机器人
- #阿里云 - 阿里云产品/服务
- #分析报告 - 各类分析报告
- #开发 - 技术开发
- #对比推荐 - 工具对比和推荐
- #配置指南 - 配置和教程

## 📝 使用说明

1. **新收集的内容** 先放入 **Inbox**
2. **定期整理** Inbox，分类到对应文件夹
3. **项目完成后** 移入 **Archives**
4. **每天** 在 **Daily** 创建日记

## 🔄 自动化

- GitHub Trending 日报：每天凌晨 4 点自动生成到 `GitHub-Trending/` 目录
- 每日笔记：在 `Daily/` 目录按日期创建
