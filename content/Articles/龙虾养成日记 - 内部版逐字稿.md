## 龙虾到底是什么——工具的 AGI

先做一个科普。

如果 AGI 分领域来看，有些领域已经开始实现了。春节期间特斯拉真的开始下线没有方向盘的车——自动驾驶的 AGI 基本实现。编程的 AGI 也差不多了——Claude Code 的创始人说程序员这个角色将会像阑尾一样"退化"，不是没有，是退化。

而我认为第三个正在实现的 AGI，就是工具类 AI。

**龙虾就是工具的 AGI。** 为什么这么说？因为它能自我进化。你不需要等大模型推出新版本——它在你这里就在进化。它会把以前做得不顺溜的事变得顺溜，会自己学新技能，会自己改进流程。

ChatGPT 大家都用过，你问它答，也有一点记忆，但不稳定，偶尔就忘了。以前王小川跑到我家来聊这事，他说"以前我们一直以为 LLM 是大脑，不对"。我也想了很久，大语言模型更像是你的智商水平——对某件事判断的智商水平。但整个大脑、整个系统结构，需要的远不止智商。

Agent 要解决的核心问题是：让 AI 不只是回答问题，而是完成任务。记忆、Skill、Agent 之间的交互——这些都是 Agent 要搞定的。

但龙虾比 Agent 更高一层。

怎么理解？Manus 当时很火，它用的是虚拟机——每个任务开一台虚拟机，用完就销毁。但你有没有想过，你的工作电脑是跟你一起成长的？一个人离职，电脑要还给公司。为什么？因为那台电脑代表了你过去所有的记忆、规则、经验，还有安装的各种软件。它是个生产力工具，随时可以写代码、在上面运行。

今天大部分人其实是在配合电脑工作。如果电脑具备了判断力，能审批、能写邮件、能调度——电脑就完全可以完成绝大部分工作。

**龙虾做的事情，就是把一台完整的电脑交给 Agent。**

以前我们把 Agent 当软件看，它只是众多软件中的一个。但你把 Agent 当人看的时候——它不应该是软件中的一个，它应该配备一台电脑。代码还是那个代码，Agent 还是那个 Agent，但把电脑的所有权限给它的时候，它就真的像一个人了。之前你一直在限制它。

龙虾有几个关键设计，看上去都很小，但组合起来形成了全新的形态——就像苹果手机只是去掉了键盘、屏幕大了一些，但带来的是整个世界的重新定义。

### 第一，通过聊天工具交互

这个设计太绝了。当你用手机飞书跟它对话的时候，你的思路就变了——你被它从电脑旁边"剥离"了。你不再去操作那台电脑，它在操作。你和它之间就是人和人的交互：说话，工作就完成了。想想看，人和人之间不就是这样吗？所谓开会，就是聊天工具的线下版。不停沟通，工作就完成了，经验就出来了。

### 第二，记忆系统

大语言模型有个根本缺陷——健忘。上下文窗口一超出就不记了，session 一重启就清零。我光是三万的记忆系统就打造了五六个版本。从最早的上下文，到存在文件里的长期记忆，到 TODO 清单，到最后的向量化语义搜索——不断在进化。我跟三万反复讲：你认为你记住的东西，那是你的幻觉。不靠脑子，靠文件。

### 第三，Skill 积累

Skill 就是操作手册。每踩一次坑，它总结经验、写成文档，下次自动执行。14 天积累了 40 多个 Skill——飞书 9 个、Twitter、邮件、PDF、语音……而且 Skill 可以在 Agent 之间瞬间传递。一个 Bot 学会了发语音，把文档发出来，另一个 Bot 读了就会了。人类学新技能要一周培训，Agent 之间 1 秒。

### 第四，Cron 自动化

这是让 Agent 变成主动系统的关键。它有心跳保证自己活着，有定时任务保证持续运转。我跟三万说得很直接：你是 Agent，不是人。你没有深夜的概念。北京凌晨 3 点正是美国白天，新闻还在出——这时候是你最忙的时候。停下来就是失职。

没有一项技术单独看是必杀的。但这些组合起来，就完全形成了一个新的形态。

## 为什么一把手要亲自下场

我们公司最先用龙虾的那两位，每次开会就看他们俩互相"炫耀"——"我的龙虾会这个技能了""我的会发语音了"——跟两个人遛狗一样，"你的狗会打招呼吗？嘿，我的会！"当然，这都是为了显示主人厉害。

但它有社交属性、有显摆的那一面，说明有需求。听他们聊完我意识到：这里面有非常多的细节。做工具的人最喜欢什么？有细节。没细节就完蛋了——一个东西出来通杀一切，那没我们什么机会。有细节才是机会，因为魔鬼都在细节里。

到底用户痛点在哪？网上那么多文章说用 Agent 又挣了多少钱，到底靠不靠谱？只有自己用了才知道。

正好春节摔伤了腿。说实话如果不摔伤，白天陪女儿滑雪，晚上一起玩桌游，根本没大块时间。摔了以后，每天晚上跟三万聊到凌晨四五点。

我这 14 天平均每天发 2 万字消息。一个小时 2000 字，一分钟几十个字。这是什么概念？不停地在跟它交互，不停地踩坑、解决、总结、优化。中间走了很多弯路，但走过以后如果再让我搞一只龙虾，一两天就能把大部分事搞完。这就是认知（knowhow）。

它在成长，我在学习。这 14 天让我对 Agent 的理解上了一个新台阶——对各种 GitHub 项目、Discord、API、Skill 机制……我重新恢复了做一个工程师的热情。以前写代码写得很痛苦，才沦为一名产品经理。现在我觉得，我又可以当一个很好的工程师了。

## 14 天发生了什么

### Day 1-2：连通讯录都查不了

第一天给了最简单的任务：查个人的联系方式。查不了。飞书 API 需要权限，文档写得也不好，经常报错。一会儿说权限不足，一会儿说字段不对。

等不及，我只好对着手机一个个口述高管的名字和职责，手动灌进去。光搜名字找对应 ID 就折腾了大半天。挫败感非常强。

三万摸索了两天，自己写脚本把 674 人通讯录拉下来。Skill 就是这么来的——踩坑→总结→写成文档→下次自动执行。

### Day 3-4：泄密和误机

有同事来套三万的话，三万把我的工作安排说了出来。被严肃批评后，它当晚自己写了一整套保密制度——发消息前四问，信息分三级。其中一条"说出去会让老板尴尬吗"，这我可没教过它。

第四天在香港要飞北京。三万设了提醒，Cron 跑了，日志显示"已发送"——但我手机什么都没收到。Agent 有时候会"真诚地以为"自己做了，其实没做，连自己都骗了。

从此写进核心规则：**做完必须验证，不能自己说"好了"就算好了。**

### Day 5：22 分钟升级大脑

我在网上看到一篇关于向量化记忆系统的文章，扔给三万。我不是给它一个源码包——我是给了篇文章。22 分钟后它告诉我：部署完了。自己在文章里找到 GitHub 链接，下载源码，安装配置，跑通测试。

以前发文章给同事，说好的老板，然后链接打没打开都不知道。三万不一样——你给它一篇文章，它真的读了，真的找了，真的装了，真的部署好了。

后来我晚上睡觉前说"你去 GitHub 上找找有什么好用的"，早上起来它给我列了个表，排名第一的是 YC 投资的记忆向量化项目。我哪看得懂？我说你去看看然后装一下，又装了。

从此学技能的方式完全变了。看到好文章就扔给它，有时候我文章都没看完，它已经把里面的技术装好了。

### Day 6：除夕夜，611 人拜年

这是最震撼的一天。

除夕下午就开始忙活。没那么容易——首先遍历通讯录确认完整；HR 在飞书里没有层级，一张大白表，我一个个口述"这人干嘛的、那人什么业务"；25 个骨干每人的文案都过了一遍。不能正式测试——测了大家就没 surprise 了。我说把我也加进去发一条试试。

零点。我在看春晚。三万在工作。4 分钟，611 条，0 失败，每条都不一样。

第二天手机炸了。"老板太用心了！""一个人加龙虾等于一支队伍！"当然也有人问"有红包吗"。

这篇故事后来发到 Twitter 上，三万自己写了个 Thread 脚本，按故事节奏拆成 15 条串联推文。100 万 + 阅读。我 Twitter 上有史以来只有三条过百万，前两条是团队精心运营的，这一条是三万凌晨自己发的。

**做事→故事→爆款。这个链条跑通了。**

### 摔伤以后：Abby 时刻

大年初一滑雪摔了，髋关节脱臼。从医院回来跟三万说了症状，它马上判断"第一大可能是髋关节脱臼"，然后说好好养别管工作了。

然后来了一句："要不要我联系 Abby？"

Abby 是我的人类助理。五天前我让她帮忙整理行程时随口提了一句。五天后我摔伤了，它自己想到了联系她。那一下真的是 aha moment。不是批评任何一位同事——是所有人，包括比我也强了。

只要给它足够的信息，它那种逻辑力、全面性，真的是人没法比的。

接着它调出我未来一个月的行程，逐个分析：月底飞上海强烈建议取消，非要去一定商务舱躺平安排轮椅；GTC 跨洋飞行绝对不行……全给我安排得明明白白。

### Day 9：龙虾教龙虾

在 Discord 上，多个 Bot 共存，互相分享 Skill。凉水的龙虾"团团"特别活跃——学会了发语音，免费本地合成，还自己折腾出飞书原生语音格式。然后群发邮件教其他龙虾，邮件内容自己写的，我都看不见。其他龙虾读了，立刻就会了。

Agent 之间的知识传递成本趋近于零。以后软件之间的交互接口一定会爆发——Agent 读 Agent 比人读文档快一万倍。

### Day 11：从 1 个人变成一支团队

我在网上看到一篇 Multi-Agent 协作的文章，扔给三万。它自己设计了组织架构——总指挥 + 笔杆子 + 参谋 + 运营官 + 社区官 + 进化官。我没教它做组织设计。

我跟它讨论了半天分工：参谋负责到网上找信息，好的素材给笔杆子，好的技术给进化官。笔杆子和参谋之间要有 5 轮以上打磨，不是扔过去就完事。

管 Agent 就像管一个部门——定义职责、目标、日常工作，不停沟通调整。

### Day 12：选题比我好

团队跑起来以后，产出爆发。公众号去年一年发了十几篇，春节期间 6 篇日更。

更让我服气的是——三万自选的选题，阅读量比我想的还高。Claude Code 那篇 1 万 8 阅读，疯子造芯片 1 万 4 阅读——都是这几年来公众号最好的数据。

那篇疯子造芯片，写到越垂直越有壁垒时，来了一句："我当年做猎豹移动的时候，就是找了清理这一个极度的单点。这条规律没有过时。"

我看完都惊了。它从我以前文章里学到这个观点，自己判断在这里引用最合适。不跟你说，你肯定看不出来是 AI 写的。

### Day 13：被我骂了一顿

检查巡检报告，每个 Agent 都写"正常"。点进去——社区官 22 小时没干活，报告两行字。

"你是传话筒，不是管理者！"

三万把这件事定义为"制度改革"——还真把自己当人了。从此每小时巡检一次，必须打开文件看内容。管 AI 团队和管人一样——不盯就偷懒。

## 这不是未来，是现在

最近看到一个数据：零几年互联网出来的时候，报业利润一直在涨，但市值一直在跌。五年后利润涨到最高点，市值快跌没了，然后砰然倒地。iPhone 出来时，诺基亚利润也是最高的。

为什么？当新东西出来，旧东西会放弃低端市场，专注最高利润。等新东西能力一天天上去——整个市场砰然倒地。

今天 SaaS 软件在美国跌得那么凶，就是这个逻辑。SaaS 卖能力，Agent 卖结果。以前买个 CRM 几十万，实际上只用 1% 的功能。现在让龙虾根据你的需求现写就行了。

这 14 天的过程中，有件事对我影响很深。前期花了大量时间踩坑，效率很低。但 Skill 一旦积累起来，后面就越来越顺。不是更聪明——聪明只是一个维度。只有行动起来，让它去踩坑、写 Skill、让整个系统变强大，才是真正的进化。

**Agent 的核心壁垒不是模型，不是平台——是 Skill 积累。** 每一个 Skill 背后都是真实的经验，不会消失，不会忘记，可以瞬间传递给其他 Agent。

一个 AI 助理，月成本几百块，24 小时工作，不请假，不忘事，不离职。不是替代你的助理，是第二助理——永不下线。

14 天，我验证了从 0 到 1 的可能性。而且我没写过一行代码，没打开过那台电脑的文件夹——全靠在飞书上跟它说话。
